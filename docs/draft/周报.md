# 年终总结大纲

## 个人总结

### 1、工作成果
#### 1.1 项目投入成果

#### 1.2 云南用途管制-等保改造成果
对系统进行了完整的密码改造，包括密码传输加密，数据库加密，数据完整性签名，双因子认证，以及服务器安全改造等内容。

#### 1.3 云南用途管制-全面优化成果
通过从前端，到后端，再到查询，以至服务器部署架构全面改造，对系统进行完整优化。

#### 1.4 云南用途管制-一体化平台迁移改造成果
为保障项目长远运行发展，项目产品团队支援云南，对一体化平台的迁移应用升级，目前已经打通了整体的工作流，采用最新的产品以及更新的应用构建方式，为用途管制升级做好前置准备。

#### 1.5 拉萨一体化-应用支援成果
在上线前的关键时期，作为主力支持拉萨项目开发3-4个月，直至推至系统上线。期间除了处理上线初期系统大量存在的细节问题外，还支持了核心的流程业务开发，地方特色的会审业务实现，在线拟文，外部对接，批后监管等模块的开发。同时也给不熟悉数慧产品的交付人员讲解了产品的基本使用，以及流程建模的基础知识内容，后逐渐进行远程支持。

### 2、个人收获
- 新架构的产品使用和基于产品的开发能力。
- 应用优化能力。
- AI模型使用能力。

### 3、 不足与展望
#### 3.1 不足
1、任务管理：工作分配不明确，导致工作干的杂，一些时间用在运维上。  
2、项目管理：组织能力欠佳，有时无法和交付人员合理的分配工作和交流。  
3、技术深度：部分技术内容深度还是不够，很多业务需求的实现不够优雅。
#### 3.2 展望
1、技术深度，在流程引擎方面继续深入，成为懂流程，会设计流程的团队成员。
2、项目管理：提高项目管理要求，严格把控代码规范质量，致力于项目代码方面高质量。
3、自动化测试能力提升，现阶段任务开发完毕后，绝大部分是由人工测试，准确度和精细度不高。
4、团队提升：开展一些项目内部的培训，如如何精准的定位问题，更完整的进行系统测试，查看功能接口用时，服务器监控以及日志基本查看等技术，打造优秀团队。

### 4、对组织的期望或建议
1、团队内部交流欠缺，目前还停留在单纯的工作和事项交汇中，推荐增加团队内部支持，心得交流等活动。  
2、优化项目管理模式，现有模式下是依靠每周任务。  
3、新产品培训，内部产品培训停留在表面。例如数慧产品的已存在系列视频，但是真正项目开发中，实施人员遇到的问题很多，一部分是在培训中提及不足，一部分是学习人员未能理解。系列教程需要优化完善。
