## OCR文件材料同步接口
### 目的
该接口用于对文件材料执行同步OCR分析。它会在单个请求-响应周期内处理上传的文件，并等待分析完成后返回提取出的文本内容。
### 接口详情
**方法**: POST
**URL**: `{{systemURL}}/api/ocr/file/material/sync/{materialId}`
---
### 路径参数
| 参数         | 类型  | 是否必填 | 描述                          |
| ---------- | --- | ---- | --------------------------- |
| materialId | 整数  | 是    | 要分析的文件材料的唯一标识符。示例：`8779164` |

### 查询参数
| 参数             | 类型  | 是否必填 | 默认值            | 描述                                |
| -------------- | --- | ---- | -------------- | --------------------------------- |
| maxWaitSeconds | 整数  | 否    | 等待OCR分析完成的最大时间 | 示例：`300`（5分钟）。如果分析耗时超过此值，请求可能会超时。 |

### 响应
#### 成功响应 (200 OK)
**JSON**
```json
{
  "success": true,
  "msg": "成功",
  "data": "# 从OCR分析中提取的文本内容...",
  "code": 200
}
```
**响应字段说明：**
*   **success** (布尔值): 指示操作是否成功。
*   **msg** (字符串): 状态消息（成功时通常为"成功"）。
*   **data** (字符串): OCR分析提取出的文本内容，使用Markdown格式。
*   **code** (整数): HTTP状态码。
